# Fixes & Stabilization Log

## Release Candidate 1.0

### Pydantic Validation Errors Fixed
- **Problem:** `DecisionOutput` raised `ValidationError` due to missing required `List[str]` fields when `None` was implicitly generated by the mock models.
- **Solution:** Updated `backend/decision_engine/decision_schemas.py` using `Field(default_factory=list)` ensuring safe initialization of arrays and preventing `500` server crashes on the `/deals` endpoint.

### Deal ID Prefix Conflict Resolved
- **Problem:** Frontend paths requested `GET /deals/deal-1` while the FastAPI endpoint strictly enforced `deal_id: int`. This triggered unhandled `422 Unprocessable Entity` parsing errors.
- **Solution:** Refactored FastAPI routing to dynamically accept string IDs, utilizing `.replace("deal-", "")` safely in the controller. This completely unblocked all analytical engine routes.

### React / Next.js Stability Improvements
- **Problem:** Total app crashing when `fetch` promises were rejected (e.g., if the backend was offline or returned a `404`).
- **Solution:** Designed and implemented a global React Error Boundary (`frontend/app/error.tsx`). Handles backend disconnects gracefully by rendering a "System Offline" component with retry controls.
- **Problem:** Visiting invalid deals rendered broken UI components.
- **Solution:** Engineered a robust `EmptyState` component for the Deal Layout protecting the user experience.

### TypeScript Compilation Patch
- **Problem:** Build failed on `npm run build` because of invalid syntax nested inside raw `fetch` string interpolations.
- **Solution:** Repaired all `page.tsx` backend fetch calls.

### Seed Consistency
- **Problem:** Duplicated seed deals ("PawPulse" instead of "VetPulse AI").
- **Solution:** Re-engineered the seed data flow (`backend/seed/seed.py`) to generate exactly 5 distinct portfolio records that maintain consistency across application restarts.
